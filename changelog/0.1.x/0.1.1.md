# 0.1.1 — 2026-06-01

Initial public release of `@cyanheads/met-museum-mcp-server`.

## Added

- **`met_search`** — keyword search across 500K+ Met objects with filters: `hasImages`, `isPublicDomain`, `isHighlight`, `medium`, `departmentId`, `geoLocation`, and date range (`dateBegin`/`dateEnd`).
- **`met_get_object`** — batch fetch full artwork records (title, artist, dates, medium, dimensions, provenance, image URLs) for up to 20 object IDs per call; partial results returned when some IDs fail.
- **`met_list_departments`** — return all 19 curatorial departments with their numeric IDs for use as `met_search` `departmentId` filters.
- **README** — install badges (Claude Desktop `.mcpb`, Cursor, VS Code), tool reference table, usage examples.
- **`manifest.json`** — MCPB bundle for one-click Claude Desktop install.
- **`bunfig.toml`** — Bun configuration for `bun audit` and `bun publish`.
- **`docs/tree.md`** — auto-generated directory structure.

## Changed

- **`server.json`** `runtimeHint` corrected from `"node"` to `"bun"` on both package entries.
- **`server.json`** `name` updated to reverse-DNS format `io.github.cyanheads/met-museum-mcp-server`.
- **`.env.example`** populated with server-specific vars (`MET_BASE_URL`, `MET_REQUEST_TIMEOUT_MS`, `MET_BATCH_CONCURRENCY`).
- **CLAUDE.md / AGENTS.md** trimmed to server-specific content (scaffold boilerplate removed; tool pattern updated to reflect actual implementation).
- **Dockerfile** `LABEL` description and `org.opencontainers.image.source` populated.
- **Collection count** updated from 470K+ to 500K+ across all descriptive text to match the live Met API total.

## Fixed

- **`met_search` / `met_get_object`** `ctx.recoveryFor()` call — spread into object literal removed; argument passed directly as intended by the framework API.
- **`met-search.tool.ts`** optional filter fields no longer double-wrapped with `?? undefined` (already optional at the type level).
- **`src/services/met/types.ts`** duplicate `isTimelineWork_duplicate` field removed from `RawObjectRecord`.
